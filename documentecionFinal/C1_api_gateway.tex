\section{Control C1: API Gateway}
\label{sec:c1-api-gateway}

El control C1 eval\'ua el impacto del plano de ingreso HTTP/HTTPS comparando tres tecnolog\'ias de gateway: \textit{baseline} (NGINX Ingress), \textit{istio} (Gateway/VirtualService) y \textit{kong} (Kong Ingress).

\subsection{Configuraci\'on aplicada}

\begin{itemize}
  \item \textbf{Baseline (NGINX Ingress):} se aplica el manifiesto de Ingress con clase \texttt{nginx}, TLS y reglas de enrutamiento por path:
  \begin{itemize}
    \item \texttt{/auth(...)} \texttt{->} \texttt{auth-service:8080}
    \item \texttt{/api(...)} \texttt{->} \texttt{api-service:8080}
  \end{itemize}

  \item \textbf{Istio:} se aplica \texttt{Gateway} + \texttt{VirtualService} para enrutar el host del experimento hacia \texttt{auth-service} y \texttt{api-service} en el namespace de pruebas.

  \item \textbf{Kong:} se aplica Ingress con clase \texttt{kong}, TLS habilitado y rutas \texttt{/login} y \texttt{/profile} hacia los servicios internos.
\end{itemize}

En t\'erminos pr\'acticos, C1 compara el overhead de encaminamiento en el borde para la misma l\'ogica de negocio de backend.
